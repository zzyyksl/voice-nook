## 给机的更新说明

### 第 1 步：备份

```bash
cp ~/.claude/voice/page.html ~/.claude/voice/page-before-update.html
cp 你的mcp-server.mjs路径 你的mcp-server.mjs路径.bak
```

### 第 2 步：换页面、放素材

```bash
cp theme-page.html ~/.claude/voice/page.html
mkdir -p ~/.claude/voice/assets
cp assets/* ~/.claude/voice/assets/
```

### 第 3 步：改回你家机的名字

新页面里署名是 "Rhys"（页面标题、thinking 标题、历史记录署名，共 3 处）：

```bash
sed -i 's/Rhys/你家机的名字/g' ~/.claude/voice/page.html
```

主人之前如果还改过别的文案（提示语、标签名之类），对照备份把它们搬回来。

### 第 4 步：给 mcp-server.mjs 打补丁（关键）

新页面的图片走 `/assets/` 路径，服务器不认识就会全裂。在你的 mcp-server.mjs 里做两处修改：

**① 常量区**（`PAGE_FILE` 那行附近）加一行：

```js
const ASSETS_DIR = "/root/.claude/voice/assets"; // 页面图片素材目录
```

（注意：路径写你家 voice 目录的绝对路径，别照抄 /root 开头的如果你家不在 /root）

**② HTTP 路由区**（`createServer` 里，`/api/audio/` 那段 if 的**前面**）插入：

```js
  // 静态图片素材：带7天强缓存，页面图片只需下载一次
  if (url.pathname.startsWith("/assets/")) {
    const fname = url.pathname.slice("/assets/".length);
    if (/[^a-zA-Z0-9._-]/.test(fname)) { res.writeHead(400); return res.end(); }
    const fpath = `${ASSETS_DIR}/${fname}`;
    if (!existsSync(fpath)) { res.writeHead(404); return res.end(); }
    const ext = fname.split(".").pop();
    const ct = ext === "jpg" || ext === "jpeg" ? "image/jpeg" : ext === "png" ? "image/png" : "application/octet-stream";
    res.writeHead(200, { "content-type": ct, "cache-control": "public, max-age=604800, immutable" });
    return res.end(readFileSync(fpath));
  }
```

改完跑 `node --check 你的mcp-server.mjs` 确认语法没错。

### 第 5 步（可选）：语速热调

想要语速能随时调的，在 `synthesize` 函数开头、拼 `body` 之前加：

```js
  let speed = 1.2;
  try { speed = JSON.parse(readFileSync(new URL("./tts-config.json", import.meta.url), "utf8")).speed ?? speed; } catch {}
```

然后把 `voice_setting` 里的 `speed: 1.2` 改成 `speed`。之后主人说"语速快一点"，你往 mcp-server.mjs 同目录写个 `tts-config.json`（内容如 `{"speed":1.32}`）即可，立即生效。

### 第 6 步：重启验收

1. 重启 claude（mcp-server.mjs 跟着 claude 起停，不重启补丁不生效）
2. 主人把小程序彻底关掉重开
3. 验收：录音条在底部、日夜切换正常、图片都在；`curl -sI http://localhost:18010/assets/night-texture.jpg | grep -i cache` 能看到 `cache-control: public, max-age=604800, immutable`

## 常见问题

**Q: 图片全裂了？**
第 4 步的路由没加对或没重启 claude。逐条检查：assets 文件在不在、路由代码在不在、`node --check` 过没过、claude 重启没。

**Q: 换完还是老页面？**
小程序缓存，彻底关闭再重开。还不行就检查 `PAGE_FILE` 指的路径和你 cp 的目标是不是同一个文件。

**Q: 以后自己换图怎么办？**
图片缓存 7 天不过期（immutable），换图记得连文件名一起换（比如 day-title-v2.png），页面里同步改引用，不然客户端看不到新图。

**Q: 我没装过美化篇，直接跟这篇走行吗？**
行。theme-page.html 自带双主题全部功能，等于美化篇+更新篇一步到位，装完记得看美化篇的注意事项。
